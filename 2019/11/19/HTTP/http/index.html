经典的五层模型

Http协议的发展史

http0.9:只有一个命令GET，没有header等描述数据的信息，服务端发送完毕就关闭tcp协议
http1.0:
*增加了请求命令（GET、POST、HEAD）
*status code
*多字符集支持
*权限
*缓存
*内容编码
*多部分发送

http1.1:
*增加请求命令（option、put、patch、delete、trace、connect）
*持久链接
*增加host
（允许复用TCP链接）

http2.0:
*二进制传输
*信道复用
*分帧传输
*server push

TCP三次握手

第一次握手：客户端发送SYN报文
第二次握手：服务端回传SYN+ACK报文
第三次握手：客户端回传ACK报文

三次握手：确保信息传输的可靠性

四次挥手

第一次：客户端发送FIN seq
第二次：服务端回传ACK seq ack=u+1
第三次：服务端发送 FIN ACK seq ack
第四次：客户端回传ACK seq ack

进行四次挥手的原因：因为数据传输结束后发出链接释放的通知，待对方确认后进入半关闭状态，当另一方也没有数据再发送的时候，则发出链接释放通知，对方确认后就完全关闭了tcp链接

http报文

*亲求报文
*响应报文

http请求方法：

*GET：请求获取Request-url所标识的资源
*post：在request-url标识的资源后附加新的数据
*HEAD：请求获取由Request-url所标识的资源的响应消息报头
*PUT：请求服务器存储一个资源，并用Request-url作为标识
*DELETE：请求服务器删除Request-url所标识的资源
*TRACK：请求服务器回送收到的请求信息，用于测试或诊断
*CONNECT：http/1.1协议中预留给能够链接改为管道方式的代理服务器
*OPTION：请求查询服务器的性能，或者查询与资源相关的选项和需求

http状态码：

1xx：表示请求已接收，继续处理
2xx：成功-表示请求已成功接收
3xx：重定向-要完成请求必须进行下一次的操纵
4xx：客户端错误-请求有语法错误或请求无法实现
5xx：服务器错误-服务端无法实现合理的请求

注意：
301 永久跳转（走缓存）
302:临时跳转

跨域（cors）

浏览器的同源策略限制了跨域请求资源
scrip可以允许跨域

跨域的限制：
*默认跨域允许的方法只有GET、POST、head，其他方法不允许

*默认允许Content-type以下3个，其他预请求验证，通过才能发送
text/heml
multipart/form-data
application/x-www-form-urlencoded
请求头限制自定义的请求头是不允许的，预请求验证通过才能发送

cache-control

可缓存性：
*pubilc 任何都可以
*private 只有它发起浏览器可以缓存
*no-cache：去服务端验证才能缓存
*no-store：彻底不能
*no-transform：代理服务器不能改动返回内容
