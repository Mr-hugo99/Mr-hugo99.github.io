JavaScript和typescript的区别：
typescript是JavaScript的超集，拓展了JavaScript的语法，因此现有的JavaScript代码可能与typescript一起工作无需任何修改，typescript通过类型注解的静态类型检查

2.typescript区分大小写
3.分号是可选的

4.typescript与面向对象
typescript是一种面向对象的语言：
class site{
name():void{
console.log(“runoob”)}
}
var obj =new site();
obj.name;

typescript 基础类型：
any 赋予人意类型的值
number 双精度64位浮点值
元祖 元祖类型用来表示一直元素数量和类型的数组，个元素的类型不必相同，对应位置的类型需要相同
枚举 枚举类型用于定义数值集合 enum
never never 是其他类型的子类型，代表从不会出现的值

变量声明：
var 【变量名】：【类型】=值；

运算符：
位运算符：
&:AND 按位与处理两个长度相同的二进制数，
｜：OR，按位或处理两个长度相同的二进制数，
～：取反
^：异或
<<：左移把<<左边的运算数的各二进制位全部若干位，由<<右边的数指定移动的位数，高位丢弃，低位补0

lambda函数
又称为箭头函数
函数只有一行语句：
（【param1，param2，···，paam n】）=>statement；

typescript接口
接口是一系列的抽象方法的声明，是一些方法特征的集合，这些方法都应该是抽象的，需要由具体的类去实现，然后第三方都可以通过这些抽象方法调用
typescript接口定义如下：
interface interface_name{}

值得注意的一点是接口不能转换为JavaScript，因为它只是typescript的一部分

接口和数组：
接口中我们可以将数组的索引值和元素设置为不同的类型，索引值可以是数字或是字符串
interface namelist{
[index:number]:string}
var list2:namelist=[“john”,1,”bran”]
interface ages{
[index:string]:number}
var agelist:ages;
agelist[“john”]=15
agelist[2]=”nice”

接口继承：
接口继承可以说是接口可以通过其他的接口来扩展自己、typescript是可以接口继承多个接口、关键字为：extends
继承各个接口是使用的是，分隔

类的继承
typescript支持继承类，即我们在创建类的时候继承一个已存在的类，这个已存在的类称为父类，继承他的类称为子类
关键字为：extends ，子类除了不能继承父类的私有成员和构造函数，其他的都可以继承，但是typescript不支持多个继承

继承类的方法重写、关键字为super

static关键字
static关键字用于定义类的数据成员（属性和方法）为静态的，静态成员可以直接通过类名调用

typescript对象
对象是包含一组键值对的实例，值可以是标题、函数、数组、对象，

鸭子类型：
duck typing 是动态类型的一种丰额。是多态的一种形式
鸭子类型是只关注对象的行为，能做什么，而不是关注对象的所属的类型

typescript命名空间
作用：解决重名问题
typescript中命名空间使用namespace来定义：
namespace somenamespacename{
export interface ISomeInterfaceName{}
}
